# Sandbox sample assets

Sample binary files used when creating the sandbox workspace in Google Drive. These are **not** in `public/` so they are not served to the browser; they are read server-side only during sandbox creation.

## Included

- **sample.png**, **sample.jpg** – minimal 1×1 image placeholders (replace with real images from sample sites).
- **sample.docx**, **sample.xlsx**, **sample.pptx** – minimal valid Office Open XML files (generated by `node scripts/generate-sandbox-ooxml.mjs`). You can replace these with real documents from the sites below for more realistic sandbox content.
- **sample.zip** – minimal ZIP archive (generated by the same script). Used for any `.zip` entries in the sandbox hierarchy.
- **sample.pdf** – minimal valid PDF (generated by the same script). Used for all `.pdf` entries in the sandbox hierarchy.

## Replacing with real samples

You can replace any of the above with real sample files from:

- https://sample-files.com/
- https://testfile.org/
- https://file-examples.com/

Keep the same filenames (`sample.png`, `sample.jpg`, `sample.docx`, `sample.xlsx`, `sample.pptx`, etc.) so the loader finds them, or update the mapping in `sandbox-asset-loader.ts`.

Optional: add **sample.gif**, **sample.webp** for those types. If an asset is missing, Office/docs/PDF fall back to generated placeholder content; images and zip are skipped.
